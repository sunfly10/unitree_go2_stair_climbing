# 🐕 Unitree Go2 Stair Climbing: 비전 기반 자율 접근 및 제어 시스템

## 1. 프로젝트 개요 (Introduction)

본 프로젝트는 **포스텍(POSTECH) 동계 타대생 연구참여 프로그램**을 통해 진행한 사족보행 로봇(Unitree Go2) 제어 실습 프로젝트입니다. 
발전소 등 복잡한 필드 환경에서 로봇이 계단을 자율적으로 인식하고 등반하기 위한 전 단계인 '**정밀 접근 및 시선 제어 시스템**'을 구축했습니다.

기존 3D 센서 퓨전(Camera-LiDAR) 시 발생하는 보행 진동 노이즈 한계를 극복하기 위해, 화면상의 픽셀 정보를 직접 제어 입력으로 사용하는 **IBVS(Image-Based Visual Servoing)** 방식을 도입하여 제어 신뢰성을 확보한 것이 특징입니다.

### 🛠️ 개발 환경 및 하드웨어
* **OS / Middleware:** Ubuntu 22.04, ROS 2 (Foxy), Cyclone DDS
* **Hardware:** Unitree Go2 (사족보행 로봇), Intel RealSense D435i
* **Core Tech:** YOLOv8 (객체 인식), IBVS (모션 제어), Adaptive Thresholding, Kinematics 기반 Pitch 제어

## 2. 주요 기술 및 알고리즘 (Key Technologies)

### 2.1 환경 적응형 비전 인식 (YOLOv8)
* **데이터셋 최적화:** Roboflow를 활용하여 계단 전체 영역(stair)과 개별 층계(step)를 세분화하여 학습시켰습니다. 이를 통해 로봇 앞의 장애물이나 사람에 의해 계단 일부가 가려지는 상황(Occlusion)에서도 인식의 연속성을 확보했습니다.
* **적응형 임계값(Adaptive Thresholding):** 로봇의 주행 상태에 따라 YOLO 모델의 신뢰도(Confidence) 기준을 동적으로 변경하여 시스템 안정성을 높였습니다.
  * `정지/탐색(SEARCH)`: 임계값을 **0.70**으로 높여 오인식을 원천 차단.
  * `접근/이동(TRACK)`: 임계값을 **0.40**으로 낮춰, 보행 진동이나 모션 블러 발생 시에도 추적이 끊기지 않도록 주행 연속성 극대화.

### 2.2 IBVS 기반 2축(Yaw, X/Y) 접근 제어

복잡한 3D 좌표 변환을 생략하고, 화면상 픽셀 좌표 오차를 제어 입력으로 직접 사용하여 제어 안정성과 실시간성을 높였습니다.
* **조향(Yaw) 및 전진(X/Y) 제어:** 인식된 계단의 중심 좌표가 화면 중앙(640px)에 위치하도록 회전 각속도를 제어하고, 계단 하단부가 목표선에 도달할 때까지 전진 속도를 비례 제어합니다.
* **안전 정지선(Stop Line) 설정:** 기구학적 역산을 통해 계단 전방 약 21.8cm 위치에 해당하는 화면 하단 650px 지점을 최적의 도착 지점(Stop Line)으로 설정하여, 충돌 없이 안전하게 등반을 준비할 수 있는 안전 마진을 확보했습니다.

### 2.3 데드존 극복을 위한 1축(Pitch) 시각 서보잉 보완 로직
로봇이 계단에 근접할수록 목표물이 카메라 화각 아래로 사라지는 **데드존(Dead Zone)** 문제를 해결하기 위해, 화면상 픽셀 위치에 비례하여 로봇의 피치 각도를 동적으로 제어했습니다.
* **선형 보간(Linear Interpolation) 적용:** 화면상 계단의 Y좌표 위치(480px ~ 650px 구간)에 따라 목표 피치 각도를 0에서 최대 0.38 rad(약 21도)까지 실시간으로 계산하여 적용했습니다.
* **지수 이동 평균(EMA) 필터:** 계산된 목표 피치 값에 가중치(Alpha=0.12)를 두어 과거 값과 혼합함으로써, 모터의 급격한 구동을 방지하고 카메라 시야의 흔들림을 최소화하는 부드러운 모션을 구현했습니다.

### 2.4 ROS 2 및 Unitree API 제어 통합
ROS 2 노드에서 계산된 제어 명령을 로봇 하드웨어가 즉각적으로 수행하도록 Unitree 내부 API 프로토콜을 연동했습니다.
* **Pose 모드 동시 제어:** 초기화 단계에서 `1028번 API(Pose 모드)`를 강제 활성화하여, 이동(`API 1008`)과 동시에 몸통의 기울기(`API 1007` - Euler Pitch)를 실시간으로 병렬 제어하는 복합 모션을 성공적으로 구현했습니다.

## 3. 트러블슈팅 및 고찰 (Troubleshooting & Conclusion)

### 💡 3D 맵핑 노이즈 한계 극복 및 제어 방식 전환
* **Issue:** 초기에는 카메라와 라이다(LiDAR) 좌표를 통합하여 3D 월드 좌표계로 변환하려 했으나, 사족보행 로봇 특유의 강한 보행 진동으로 인해 1m 전방의 거리 오차가 10cm 이상 발생하는 Extrinsic Calibration 난항을 겪었습니다.
* **Solution:** 복잡하고 불안정한 3D 연산을 과감히 생략하고, **2D 이미지 평면의 픽셀 오차를 활용하는 IBVS 제어로 아키텍처를 전면 전환**하여 실시간성과 시스템 제어 신뢰성을 모두 확보했습니다.

### 🚀 향후 과제 (Future Works)
* 현재의 비전 기반 접근 제어를 넘어, 3개의 다리 관절 모터를 개별 제어하는 **계단 등반 알고리즘의 구체화**.
* LiDAR를 활용한 3D Point Cloud 맵핑 융합을 통한 보행 환경 인지 고도화.
* 하드웨어적 보강(등반 시 지지팔 등 보조 기구 활용)을 통한 물리적 등반 안정성 연구.
